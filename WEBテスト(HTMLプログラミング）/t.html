1. 序論：背景にある課題

災害時、避難所における「たらい回し」問題

現状の避難所運営はリアルタイムの空き状況が分からず、以下の問題が発生しています。

  • 大規模災害時、特定の避難所に人が殺到し、「満員で入れない」事態が発生。
  • 疲労困憊の中、避難者が次の場所を探してさまよい、二次被害のリスクが高まる。

情報の空白が引き起こす「負の連鎖」

2. 提案：スマート避難誘導システムの概要

本提案は、IoT技術とWeb技術を組み合わせて、「情報の空白」を解消します。

データ収集：IoTセンサー

避難所に設置されたIoTセンサー（CO2濃度、AIカメラ）が**混雑状況を自動検知**し、客観的なデータをクラウドにリアルタイム送信。

情報配信：Webマッチング

避難者のスマートフォンに、リアルタイムデータに基づき**「今、空いている一番近い避難所」**をナビゲートします。

目標：避難の「分散化」と「最適化」

3. 技術的仕組み I：データ収集（IoT/センサー）

汎用性の高い安価なハードウェアで構築します。

CO2濃度センサーによる「密集度」推定

換気状態と二酸化炭素濃度から、収容人数に対する**密度の高まりを客観的に判断**。個人特定せずプライバシーに配慮。

AIカメラによる入退場カウント

避難所の入口で入退場人数をカウントし、**正確な残りの収容可能人数**をリアルタイム算出（Deep Learning不要な軽量モデルを使用）。

ハードウェア基盤

Raspberry PiやArduinoなどの**汎用マイコン**を採用し、導入コストを大幅に抑制。

4. 技術的仕組み II：情報配信（Web/アプリ）

【リアルタイム避難所マップのアプリ画面イメージ】

避難者に必要な情報を過不足なく提供します。

Web APIによるリアルタイム連携

収集されたIoTデータは、Google Cloud Firestoreなどの**クラウドDB**に格納され、Web APIを通じてアプリに即時配信されます。

Web GIS（地図情報）活用

避難者の現在地、目的地までの経路、避難所の属性（ペット可、バリアフリーなど）を地図上に統合しナビゲーション。

情報信頼性（デマ対策）

避難所の空き状況は**センサーの機械的なデータ**に基づいており、人為的な操作やデマによる混乱を防止します。

5. 導入効果：誰に、どのようなメリットがあるか

避難者へのメリット

  • 迷子の防止： 疲労や不安の中で、**確実に入れる避難所**へ最短でナビゲート。
  • 属性マッチング： 特定のニーズ（医療、ペット）に対応した避難所をすぐに見つけられる。

自治体・運営者へのメリット

  • 物資配給の最適化： 各避難所のリアルタイム人数に基づき、水や食料の**在庫を正確に調整**できる。
  • 人員配置の効率化： 混雑状況が分かるため、職員の緊急配置も的確に行える。

6. 懸念される課題と対策（リスクマネジメント）

システム導入における主な懸念点と、その具体的な解決アプローチです。

課題① 通信遮断リスク（ネットダウン）

**対策：** 携帯回線に依存せず、災害に強い**LPWA（低消費電力広域通信）**や衛星通信など、冗長化された通信インフラをIoTセンサー側に採用する。

課題② 導入・維持コスト

**対策：** 汎用マイコンとオープンソース技術を活用し、**ハードウェアコストを最小化**。クラウドインフラは、自治体向け無料枠などを活用したSaaSモデルを提案。

課題③ 避難者によるアプリ利用の有無

**対策：** アプリだけでなく、避難所入口に設置した**デジタルサイネージ**や、地域の防災無線・Webサイトとも情報を連携し、多様な情報提供チャネルを確保する。
